## Extension 1
```
A local candle maker wants to open an online store selling their handmade candles and other artifacts, they need a database system to use as part of this online shop.
Details of customers need to be stored, products need to be stored and updated, a record of customer orders also needs to be maintained.
Design the database system for the Candle Shop.
```
### User Stories

```
As a candle maker,
So that i can sell my stock,
i want to list the products i sell.

As a candle afficianado,
so that i can smell an artisan candle,
i want to be able to make an order.

As a redistributor,
so that i can get ahold of a large amount of candles
i want to be able to order multiple products.

As a candle maker,
So that i don't get more orders than i can produce,
I want to list how many i have available.

```

## Extension 2

```
A car hire compnay with multiple locations wants to design an online system that will allow customers to book various cars,
on various dates from different locations.
Design the database system that would allow them to effectively manage their business.
```
### User Stories

```
As a customer,
So that i can get around while im on vacation,
i want to rent a car on a particular date.

As a happy camper,
so that i can go enjoy the great outdoors,
i want to rent a car that suits my needs.

As a company director,
so that i can get customers to rent more cars
I want to use the data from previous rentals to improve our service.
```

## Extension 3
```
A University library wishes to implement an online system for borrowing books and other items from the library.
Each user will be able to borrow up to 6 items at a time.
```
### User Stories
```
As a librarian,
so that students don't borrow a book over the limit,
i want to limit the number of books a student can borrow

As a student,
so that i don't waste my time looking,
i want to be able to see if a book is on loan.

As a professor,
so that i can get the newest discoveries within my field,
i want to know about the newest additions within that category and type.
```
